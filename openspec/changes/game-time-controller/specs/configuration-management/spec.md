## 新增需求

### 需求: 从文件加载配置
系统应在启动时从用户可配置的文件加载配置。

#### 场景: 启动时加载配置
- **当** 系统启动时
- **则** 它从默认配置文件位置（例如 ~/.config/game-control/config.yaml）加载配置

#### 场景: 从自定义路径加载配置
- **当** 系统启动并指定了自定义配置文件路径时
- **则** 它从指定路径加载配置

### 需求: 验证配置
系统应验证所有配置值并拒绝无效配置。

#### 场景: 有效配置
- **当** 配置文件包含有效值时
- **则** 系统成功加载配置

#### 场景: 无效的时间限制
- **当** 配置包含无效的时间限制（例如负值）时
- **则** 系统报告错误并以非零状态退出

#### 场景: 无效的重置时间
- **当** 配置包含无效的重置时间（例如 25:00）时
- **则** 系统报告错误并以非零状态退出

### 需求: 存储配置默认值
系统应为所有配置选项提供合理的默认值。

#### 场景: 配置文件缺失
- **当** 配置文件不存在时
- **则** 系统使用默认配置值

#### 场景: 配置文件不完整
- **当** 配置文件缺少某些选项时
- **则** 系统对缺失的选项使用默认值

### 需求: 支持用户自定义游戏进程的配置
系统应允许用户使用配置来标识哪些进程被视为游戏，需要被检测和控制。

#### 场景: 
- **当** 配置包含游戏进程名单时，如 `["game1.exe", "game2.exe"]`
- **则** 系统监控在该名单内的进程
