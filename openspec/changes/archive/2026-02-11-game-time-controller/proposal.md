## 背景

游戏沉迷已成为现代社会的普遍问题，影响用户的工作、学习和生活健康。现有解决方案往往过于复杂或需要付费，而简单的命令行工具可以帮助用户通过自动化控制来管理游戏时间。

## 变更内容

- **新增游戏进程检测功能**：自动识别和监控正在运行的游戏进程
- **新增时间限制控制功能**：设置全局每日游戏时长上限，当达到限制时自动终止游戏进程
- **新增每日配额重置机制**：每天早上8点自动重置可用游戏时长
- **新增配置管理功能**：允许用户自定义时长限制、检测规则和重置时间
- **新增状态监控功能**：实时显示当前游戏时长使用情况和剩余时间

## 能力

### 新增能力
- `game-process-detection`: 游戏进程识别、监控和管理能力
- `time-limit-control`: 游戏时长限制、配额跟踪和强制终止能力
- `daily-quota-reset`: 每日配额自动重置和定时任务调度能力
- `configuration-management`: 应用配置加载、验证和持久化能力
- `status-monitoring`: 游戏时长使用状态显示和日志记录能力

### 修改的能力
- (无现有能力需要修改)

## 影响

- 这是一个全新的 Go 项目，不涉及现有代码或 API 的修改
- 支持 Windows 系统
- 依赖外部进程管理库（如 `os/exec`）进行进程操作
- 需要 Windows 管理员权限来终止其他进程
- 可能使用 JSON 或 YAML 格式存储配置文件
- 不需要网络通信或数据库依赖
