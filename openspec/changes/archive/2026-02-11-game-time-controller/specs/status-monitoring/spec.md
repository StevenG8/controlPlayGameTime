## 新增需求

### 需求: 显示当前游戏使用时间
系统应显示当天的累计游戏时间。

#### 场景: 显示累计时间
- **当** 用户请求当前状态时
- **则** 系统显示当天的累计游戏时间

#### 场景: 以适当格式显示时间
- **当** 显示累计时间时
- **则** 系统以小时和分钟格式显示时间（例如 "1h 30m"）

### 需求: 显示剩余时间
系统应显示当天剩余的可用游戏时间。

#### 场景: 显示剩余时间
- **当** 用户请求当前状态时
- **则** 系统显示剩余的可用游戏时间

#### 场景: 显示零剩余时间
- **当** 每日限制已超限时
- **则** 系统显示 "0h 0m" 作为剩余时间

### 需求: 显示活跃游戏进程
系统应显示当前活跃的游戏进程列表。

#### 场景: 显示活跃进程
- **当** 游戏进程当前正在运行时
- **则** 系统显示活跃游戏进程列表及其名称和启动时间

#### 场景: 没有活跃进程
- **当** 没有游戏进程当前正在运行时
- **则** 系统显示指示当前没有游戏运行的消息

### 需求: 显示下次重置时间
系统应显示距离下次每日配额重置的时间。

#### 场景: 显示距下次重置的时间
- **当** 用户请求当前状态时
- **则** 系统显示直到下次配额重置的持续时间

### 需求: 记录游戏事件
系统应记录重要的游戏相关事件以用于审计。

#### 场景: 记录游戏启动
- **当** 游戏进程启动时
- **则** 系统记录进程名称和启动时间

#### 场景: 记录游戏停止
- **当** 游戏进程停止时
- **则** 系统记录进程名称、停止时间和会话持续时间

#### 场景: 记录配额重置
- **当** 每日配额重置时
- **则** 系统记录重置事件和时间戳

#### 场景: 记录限制超限
- **当** 每日时间限制超限时
- **则** 系统记录限制超限事件并终止游戏进程
