## 新增需求

### 需求: 检测运行中的游戏进程
系统应根据可配置的游戏进程名单，持续监控并检测正在运行的游戏进程。

#### 场景: 检测到游戏进程
- **当** 一个与配置的游戏进程名称模式匹配的进程正在运行时
- **则** 系统将其识别为游戏进程并记录其启动时间

#### 场景: 检测到多个游戏进程
- **当** 多个与游戏进程模式匹配的进程正在运行时
- **则** 系统识别所有匹配的进程并单独跟踪每个进程

### 需求: 终止游戏进程
系统应在每日时间限制超限时终止游戏进程。

#### 场景: 终止单个游戏进程
- **当** 每日游戏时间超限且有一个游戏进程正在运行时
- **则** 系统优雅地终止该游戏进程

#### 场景: 终止多个游戏进程
- **当** 每日游戏时间超限且有多个游戏进程正在运行时
- **则** 系统终止所有游戏进程

### 需求: 跟踪游戏进程运行时
系统应跨多个会话跟踪游戏进程的累计运行时间。

#### 场景: 从单个会话累计运行时间
- **当** 游戏进程运行然后停止时
- **则** 系统记录该会话的持续时间

#### 场景: 从多个会话累计运行时间
- **当** 游戏进程在同一天内多次运行时
- **则** 系统在所有会话中累计总运行时间

#### 场景: 单个会话运行多个游戏，只计算一次时间
- **当** 同一时间有多个游戏进程同时运行时
- **则** 系统只计算一次时间，避免重复计时

### 需求: 监控进程状态
系统应实时跟踪受监控游戏进程的状态。

#### 场景: 进程启动
- **当** 游戏进程启动时
- **则** 系统检测新进程并开始跟踪

#### 场景: 进程停止
- **当** 游戏进程停止（自然终止或被终止）时
- **则** 系统检测进程终止并完成运行时间跟踪
