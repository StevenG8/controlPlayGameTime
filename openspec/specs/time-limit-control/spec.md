## 新增需求

### 需求: 强制执行每日时间限制
系统应对所有游戏强制执行可配置的每日游戏使用时间限制。

#### 场景: 时间限制未超限
- **当** 累计游戏时间低于每日限制时
- **则** 游戏进程继续正常运行

#### 场景: 时间限制超限
- **当** 累计游戏时间达到或超过每日限制时
- **则** 系统终止所有游戏进程，如果后续发现游戏进程又重新启动，应继续终止

### 需求: 计算剩余时间
系统应计算并显示剩余的可用游戏时间。

#### 场景: 开始时计算剩余时间
- **当** 今天还没有玩游戏时
- **则** 剩余时间等于配置的每日限制

#### 场景: 使用过程中计算剩余时间
- **当** 今天已经玩了游戏一段时间时
- **则** 剩余时间等于每日限制减去累计游戏时间

#### 场景: 剩余时间为零
- **当** 累计游戏时间等于每日限制时
- **则** 剩余时间为零

### 需求: 接近时间限制时警告
系统应在接近每日时间限制时向用户发出可见弹窗警告，并且每个警告时间点在单个配额日内只触发一次。

#### 场景: 达到首个警告阈值
- **当** 剩余时间低于首个警告阈值且高于最后警告阈值
- **则** 系统向用户显示首个警告弹窗

#### 场景: 达到最后警告阈值
- **当** 剩余时间低于最后警告阈值
- **则** 系统向用户显示最后警告弹窗

#### 场景: 首个警告阈值重复轮询
- **当** 系统在同一配额日内多次轮询且剩余时间持续处于首个警告阈值区间
- **则** 系统只在首次进入该阈值区间时弹窗一次

#### 场景: 最后警告阈值重复轮询
- **当** 系统在同一配额日内多次轮询且剩余时间持续处于最后警告阈值区间
- **则** 系统只在首次进入该阈值区间时弹窗一次

#### 场景: 每日重置后恢复提醒
- **当** 每日配额重置完成
- **则** 系统清空当日警告触发状态，使新一天可再次在各阈值触发一次弹窗

### 需求: 超限后阻止游戏启动
系统应在每日限制超限后阻止新的游戏进程启动。

#### 场景: 超限后尝试启动游戏
- **当** 用户在每日限制超限后尝试启动游戏时
- **则** 系统立即终止游戏进程并显示已达到限制的消息
