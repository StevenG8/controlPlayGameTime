## 新增需求

### 需求: 分发目录提供自启动管理脚本
系统应在 Windows 分发目录中提供添加/删除自启动任务的脚本。

#### 场景: 添加自启动任务成功
- **当** 用户在分发目录执行 `add-autostart.bat`
- **则** 系统在任务计划程序中创建登录触发任务，任务动作为后台启动 `game-control.exe start config.yaml`

### 需求: 自启动可卸载
系统应支持通过脚本移除已安装的自启动任务。

#### 场景: 卸载自启动任务成功
- **当** 用户在分发目录执行 `remove-autostart.bat`
- **则** 系统删除对应任务计划项，后续登录不再自动启动控制器

### 需求: 控制器单实例运行
系统应保证同一用户会话下仅有一个控制器实例运行。

#### 场景: 自启动与手动启动并发
- **当** 自启动实例已经在运行，用户再次执行 `game-control.exe start`
- **则** 系统拒绝启动第二个实例并返回清晰提示
