## 新增需求

### 需求: 在计划时间重置每日配额
系统应在可配置的时间（默认：上午 8:00）自动重置每日游戏时间配额。

#### 场景: 计划重置发生
- **当** 系统时钟达到配置的重置时间时
- **则** 累计游戏时间重置为零，可用时间恢复为每日限制

#### 场景: 错过计划时间后重置
- **当** 系统启动时已在配置的重置时间之后
- **则** 启动时立即重置每日配额

### 需求: 跟踪重置计划
系统应跟踪下次计划的重置时间并向用户显示。

#### 场景: 显示下次重置时间
- **当** 用户查询状态时
- **则** 系统显示距离下次配额重置的时间

#### 场景: 重置后更新下次重置时间
- **当** 发生配额重置时
- **则** 计算并存储下次重置时间（24 小时后）

### 需求: 跨重启持久化配额状态
系统应将累计游戏时间和配额状态持久化以在系统重启后保持。

#### 场景: 持久化当前配额状态
- **当** 系统正在运行时
- **则** 当前累计时间和配额状态定期保存到磁盘

#### 场景: 启动时恢复配额状态
- **当** 系统启动时
- **则** 它加载持久化的配额状态并从保存的值继续跟踪

### 需求: 允许手动重置
系统应允许授权用户在需要时手动重置每日配额。

#### 场景: 手动重置命令
- **当** 授权用户执行手动重置命令时
- **则** 累计游戏时间立即重置为零
